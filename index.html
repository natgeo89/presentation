Here I need to say that you can imagine blockchain just like a type of database. But, how does a blockchain differ from a database? The answer is the way the data is structured.
Classic Database
In classic databases information is typically structured in table format that allows easy search and filter specific information. If it's necessary we can easily change data inside such database.
Blockchain
A blockchain collects information in Blocks, that hold sets of information. Blocks when filled, are chained onto the previously filled block, forming a chain of data known as the “blockchain.” We can say that all blockchains are databases but not all databases are blockchains. Blockchain have an interesting property: once some data has been recorded inside a blockchain, it becomes very difficult to change it.
So, how it works? let’s take a closer look at a block.
  1. Data
  2. Hash
  3. Hash of the previous block
PUSH Each block contains some data PUSH, the hash of the block PUSH and the hash of the previous block. The data that is stored inside a block depends on the type of blockchain.
DATA
The Bitcoin blockchain for example stores the details about a transaction, such as the sender, receiver and amount of coins.
HASH

A block also has a hash. You can compare a hash to a fingerprint. It identifies a block and all of its contents and it's always unique, just as a fingerprint.
Hash of the previous block
The third important element inside each block is the hash of the previous block. So each block links to the previous block. It is how chain of blocks is created. Let's look on example.
As you can see, each block has a hash and the hash of the previous block. So block number 3 points to block number 2 and number 2 points to number 1. But the first block is a bit special, it can't point to previous blocks because it's the first one. It calls the genesis block.
DECENTRALIZATION
Blockchain has another important property: decentralization
Traditional system
Traditional ledgers are centralized and use 3rd parties to approve and record transactions. Here in the center of picture you can imagine Bank, that stores your money and transactions. Or maybe medical clinic, that stores your medical records And so on... I think even country government takes place here.
Blockchain system
In a decentralized blockchain network, no one has to know or trust anyone else. Each member in the network has a copy of the exact same data in the form of a distributed ledger.
All the nodes in this network create consensus. They agree about what blocks are valid and which aren't. Blocks that are tampered will be rejected by other nodes in the network. So this makes blockhain is so secure.

Usage cases
Let's a bit talk about how this technology can be used in the real world?
Cryptocurrency
Let’s start with the most obvious and most popular application of blockchains. And that is cryptocurrencies. When Bitcoin launched in 2009 it allowed people to directly transact with one another without having to trust third parties like banks. Since then over thousand different cryptocurrencies have been created.
Patents
Patents. It's my favorite example. If you are a singer, or maybe writer or you are creating any another intellectual property you can encrypt your document by hash-function, receive hash and append this hash into message in bitcoin transaction. Once added, you can always prove that you created a document at a certain point in time
Voting
Digital Voting. Patents is my favorite example, but voting, I think, the most important. Voting on paper costs a lot of money, and can be easily tampered. But instead of paper, we could use blockchains to cast and store votes. Such a system would be very transparent as everyone could verify the voting count for themselves.
